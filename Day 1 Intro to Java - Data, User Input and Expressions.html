Why Program in Java?

Java is an object-oriented programming language. Object-Oriented Programming (OOP) evolved out of the need to develop complex programs in a systematic, modular approach.

  • OOP programmers create modules, called classes that can be used over and over in a variety of programs
  • A class groups related data and the instructions for performing actions on that data
  • The Java Platform runs on all major operating systems allowing Java applications to run on just about any computer (known as platform-independent applications)
Objects, Classes, Packages & Applications
  • OOP development involves selecting objects to perform tasks outlined in program specification
  • An object consists of:
    • Related data
    • Instructions for performing actions on that data
  • A class
    • Defines the type of data and actions that will be associated with an object of that class (but not the actual data for an individual object)
    • Is required to create objects
    • A class can be used to create many objects – each object will have the same type of data and possible actions, but each object maintains its own set of data.

  • A package (sometimes called a library) is a group of related classes
    • Here is an illustration of a DVD class and a CD class put into a Media Package:

  • A Java Application is a package with at least one class that contains a main() method.

  • Classes are importable - an application can use existing classes from another package by using the import keyword

Comments
  • Comments provide information about the program to the reader of the code
  • Comments have no effect on how the program runs
  • Allow the reader to quickly understand the purpose and logic behind segments of code
  • Complex applications are often developed and maintained by more than one programmer, properly commented code allows for easier modifications and can decrease the number of mistakes
  • Java programs can contain three types of comments:

    • /** */
      • Used for documentation

Code conventions are a set of guidelines for writing an application. They provide details about commenting, rules for naming methods, classes, and packages, and statement formatting. They make programs easier to read and understand. Organizations that employ programmers often not only encourage the use of code conventions, but require it.

Through this course, code conventions will be introduced as needed. It is important that you pay attention to these conventions and use them when you write your code.

  • A Program Header should begin a program.
    • The comment should include information such as your name, class name, the date, and a brief statement about the program
  • Package names should begin with a lowercase letter and then an uppercase letter should begin each word within the name, no spaces!
  • Class names should be nouns and begin with an uppercase letter and an uppercase letter should begin each word within the name, no spaces!
  • A comment block should be included before each class and method.
    • Not typically placed before the main() method
  • An open curly brace { should be placed on the same line as the class or method declaration, and the closing curly brace } should be on a separate line and aligned with the class or method declaration

Variables

  • A variable is a name for a value stored in memory
  • Variables are used in programs so that values can be represented with meaningful names
  • Variables should be used to represent values because they make code easier to read, understand and modify
Declaring Variables
  • Declaration: A variable must be declared before it is used. This is the form:
<type> <name>

<type> is the data type (determines the type of data the variable will store)
<name> is the identifier (the variable name – should be descriptive)

  • To declare a variable named length that will hold an integer (positive or negative whole number), you would type:
int length;
  • Multiple variables with the same type can be declared in a single statement, similar to:
int length, width;
Using Variables

Example 1: The RectangleArea program

Consider the following program in Java:

import simpleIO.*;

/**
* Calculates and displays the area of a rectangle.
*/

public class RectangleArea {
  public static void main(String[] args) {

    //Variable Declaration and Initialization
    int length = 10, width = 2;
    int area;

    //Processing: Use area formula
    area = length * width;

    //Output the calculated area
    Console.print("Area of rectangle: " + area);
  }
}

Some important terminology to note:

  • Assignment: an assignment statement is formed with the variable name on the left and the value on the right of an equal sign. The value on the right can be a literal (actual value, ie. 5), or another variable or expression (ie. length*width)
  • Initialization: An assignment statement can be part of a variable declaration. In this case, the variable is initialized. For instance, length and width were both initialized (assigned values) when they were declared.
  • Concatenation: To append, or concatenate, the value of a variable to a String, the + operator is used. The result is then passed into the Console.print() statement for the user to read.

It is important to realize that a variable can store only one value at any one time. For example, after the following statements execute:

int x;
x = 5;
x = 10;

the value of x is 10, because this was the last value assigned.

Constants

  • A constant is a name for a memory location that stores a value that cannot be changed from its initial assignment
  • Using constants is considered good programming style
  • Use the keyword final to define a constant, 
final int SCREEN_WIDTH = 800;
final double PI = 3.14159265358979;
  • A constant can be assigned a value only once

Naming Variables & Constants

  • Variable & constant names must begin with a letter, and may contain letters, numbers, _, and $
  • Begin variable names with a lowercase letter, and then an uppercase letter should begin each word within the name, no spaces!
  • Constant names should be all uppercase with an underscore between words if necessary
  • Don't use the same variable name with different cases.
    • For example, lastname and lastName are considered to be two different variables in Java because of the fact the letter “N” is in a different case. 
  • Don't use reserved words and keywords (they will turn into a different colour if you type them in) of the Java language.
    • Examples of reserved or keywords in Java are: if, else, do, while, continue, break, case, switch, void, null
  • Grouping variables together in a single statement is good programming style as long as variables represent related items, not just because they happen to be the same type.
  • Group all variable declarations at the beginning of a method. Follow the declarations with a single blank line.
  • Constant declarations should be grouped at the beginning of a method before any variable declarations

Data Types

Data types describe the kind of data a variable or constant can store. Java provides built-in data types, but you can also define your own data types by using classes.

Simple or primitive data types represent a single piece of information, for example, a single number or a single sequence of text. Working with primitive values is usually faster than working with complex values, because Java stores primitive values in a special way that makes memory and speed optimizations possible.

Strings: Compound Data Types

Strings are comprised of a set of characters, and therefore cannot be represented by a primitive data type. They are instead considered Compound, or Abstract Data Types (ADTs). String variables are actually instances of the String class (defined in the java.lang package), although we rarely declare them as such. Wrapper classes (Integer, Double, Boolean, etc) are also considered ADTs.

The following statements are equivalent in Java:

String course = "Computer Science";
String course = new String("Computer Science");

User Input

Just as a computer incorporates the input-processing-storage-output model, individual programs incorporate this model as well. The programs we have written so far have begun with some values (the input), have made calculations (the processing), have kept track of the results (the temporary storage) and have sent the processed data to the screen (the output). The only problem we have with these programs is that the input can't be easily changed; the values of each variable are always going to be what the programmer assigned them to be.

The user is the person using the computer program and the user needs to interact with the program in order to make the program useful. When programming, the programmer must realize that s/he is not the user of the program. The user is the person who interacts with the program for his/her own use. In order for the program to be useful, there must be a way for the user to interact with the program.

There are several ways to interact with a program, like clicking the mouse on certain parts of the program or entering information using the keyboard. In the last two activities, you have created programs that did not require the user to interact with the program at all. In this activity, you will have the user interact with the program by inputting information into it by using the keyboard.

Example 4: The RectangleArea program, revisited

Below is a simple example of the readInt method being used:

import simpleIO.*;

/**
* Calculates and displays the area of a rectangle.
*/

public class RectangleArea {
  public static void main(String[] args) {

    //Variable Declaration
    int length, width, area;

    //Program Description
    Console.print("This program will determine the area of a rectangle");

    //Input

Notice that the Console.readInt("...") part of the code is on the right side of an assignment statement. This will initialize the variable (give it a value) to the number that the user types in.  Unlike the Console.print("...") method, you would rarely ever see this method used on its own, without being assigned to a variable, used in a calculation or as an input to another method.

Operator Precedence

  • Java uses a specific order of operations based on operator precedence
  • Multiplication and division have the highest precedence, followed by addition and subtraction
  • You can alter the order of operations by using parentheses as in math!
    • Ex) 6 + 4 * 2 – 1 evaluates to 13
    • Ex) (6+4) * (2-1) evaluates to 10
  • It is considered good programming style to include brackets where there is any possibility of ambiguity or question about the expression

Example 7: Determining the Date of Easter

The date of Easter Sunday can vary widely from year to year. The Council of Nicaea in A.D. 325 decreed that Easter Sunday was to be held on the first Sunday following the full moon that occurred on or after the first day of spring. Because the date depends on solar, lunar, and calendar cycles, it is not easy to find an arithmetic procedure that will determine the correct date for any given year.

The first such procedure was determined in 1876; an adaptation of that procedure is given below. It involves a sequence of divisions that start with the year, and eventually produce a month number (3 for March, 4 for April) and the day of the month.

The Java implementation of the algorithm is shown below

// a - m - the quotients and remainders needed
int a, b, c, d, e, f, g, h, i, j, k, m, p;
int month; // Easter month
int day; // Easter day
int year; // Easter's year

// Prompt user for year
year = Console.readInt("Easter Date Calculator\nPlease enter a year");

// Calculations
a = year % 19;
b = year / 100;
c = year % 100;
d = b / 4;
e = b % 4;
f = (8 + b) / 25;
g = (b - f + 1) / 3;
h = (19 * a + b - d - g + 15) % 30;
i = c / 4;
k = c % 4;
j = (32 + 2 * e + 2 * i - h - k) % 7;
m = (a + 11 * h + 22 * j) / 451;
month = (h + j - 7 * m + 114) / 31;
p = (h + j - 7 * m + 114) % 31;
day = p + 1;

Console.print("Easter will fall on Sunday, " + year + "-" + month + "-" + day);

Type Conversion

A type conversion occurs when a value is changed into a value of a different data but compatible type.

Automatic Conversion (Coercion)

Widening Conversion - casting not needing when converting from left to right

Automatic conversion (also called Implicit Conversion) is performed by Java at run time. For example, when a value of a 'smaller' numeric type is assigned to a variable of a 'larger' numeric type, the numeric types are implicitly converted. This is also referred to as 'widening' type conversion. The following picture illustrates widening conversion:

Explicit Conversion (Casting)

A programmer can use program code to tell the compiler to treat a variable of one data type as if it belongs to a different data type. You do this by enclosing the new type in parentheses and placing the name of the variable after the parentheses.

Explicit casting is required when you are assigning a value of 'larger' data type to a 'smaller' data type. This is also referred to as 'narrowing' type conversion. The following picture illustrates narrowing conversion:
Narrowing Conversion - always cast when converting from left to right

While it is possible to convert Strings to other data types, you cannot use a casting operator to do this. Instead, you will make use of conversion methods provided by the String class, or other Wrapper classes.

Example 8: Explicit vs Implicit Conversion

A common task for programmers is to convert your double data types to int or long, or to convert your int or long data types to double. This can be accomplished in a number of ways.

double to int or long

(Explicit Conversion)
  1. If you have a value stored in a variable declared as type double, you can assign it to a variable of type int or long, but you will need to cast the variable from a double to an int.
    Observe the following code:
double num = 5.93323;
int i;
i = num;
Console.print("i = " + i);

This will cause an error since there is a possible loss of precision. Modify the code as follows:

i = (int)num;

Now the code works! Notice that it doesn't round the decimals; it simply truncates. Explicit conversion can result in a loss of precision!

  1. If you wish to round the double variable when you assign it to the int or long variable, you can use the Math.round method.
    Observe the following code:
double num = 5.93323;
int i;
i = Math.round(num);
Console.print("i = " + i);

This will not work since the rounding function for a double returns a LONG not an INT. Modify the code as follows:

i = (int)Math.round(num);
int or long to double

(Automatic or Implicit Conversion)		 If you have a value stored in an int or long variable, simply assign it to a variable declared as type double. 
int num = 525600;
double d;
d = num;
Console.print("d = " + d);

Rounding Numbers in Java

When working with floating point numbers, there is often a need to round the final answer before outputting the value to the user. There are a few different options to do this.

Option 1: Using Math.round(). This function will return an int or a long value that is rounded to the nearest whole number. You can use a combination of multiplication and division by powers of 10 to round to a certain number of decimal places.

Option 2: Use the methods in the Console class to format your values as you output them to the user. This will not change the value stored in memory, but affect how it is displayed.

Example 9: Rounding in Java - using Math.round()

The Math.round() function always rounds to the nearest whole number. See below for a reminder about rounding rules:

    long answer1;
    double answer2;

    //Rounding to a whole number
    answer1 = Math.round(204.999);
    Console.print(answer1); // displays 205
    answer1 = Math.round(204.499);
    Console.print(answer1); // displays 204

    //Rounding to a tenth
    answer2 = Math.round(204.999 * 10) / 10.0;
    Console.print(answer2); // displays 205.0
    answer2 = Math.round(204.499 * 10) / 10.0;
    Console.print(answer2); // displays 204.5

Example 10: Rounding in Java - using the Console class

The Console class has a few options for rounding numbers before they are displayed to the user:

    //Rounding to a whole number
    Console.print(204.999, 0);  // displays 205
    Console.print(204.499, 0);  // displays 204

    //Rounding to a tenth
    Console.print(204.999, 1);  // displays 205.0
    Console.print(204.499, 1);  // displays 204.5

    //Rounding within a String using roundDouble()
    Console.print("The answer is: " + Console.roundDouble(204.499, 1));  // displays "The answer is: 204.5"
