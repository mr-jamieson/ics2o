Algorithm Design

An algorithm is a set of steps for solving a particular problem. To be an algorithm, a set of rules must be unambiguous and have a clear stopping point. We use algorithms every day; for example, a recipe for baking a cake is an algorithm. Most computer programs consist of algorithms. One of the key challenges in programming is inventing algorithms that are simple, requiring the fewest steps possible.

Flowcharts are used to communicate algorithms in language-independent terms. A flowchart is a diagram using symbols and arrows which describes the solution to a computer problem in a step-by-step manner.

The first thing you need is a set of symbols you can use so that your flowcharts make sense and look organized.

Symbol Meaning
Terminator—the beginning or end of the algorithm
Process—calculation, assigning value, do something
Input/Output—data entered and stored, answer displayed.
Decision—question asked—True/False or Yes/No response only.
Connector—connects one part of the flowchart to another.
Off page connector—used if flowchart must be continued on another page.
Directional arrows—connect one symbol to another.
Flowcharting Rules
  • Each symbol has one entry and one exit point except Decision (one entry and two exits) and Terminator (one entry or one exit only).
  • Each arrow connects to each symbol or arrow.
  • Arrows do not cross.
  • Yes/no responses should face the same direction for each decision symbol.
  • All lines and symbols are drawn neatly and with precision.
  • Best Practice: have one start and one stop.

Boolean or Logical Expressions

In programming, decisions are made based on conditions. A condition takes the form of a boolean expression (sometimes called a logical expression). Every boolean expression has one of the two boolean values: true or false.

Boolean expressions can be created in the following ways:

  • boolean variables (can be assigned a value of true or false)
  • comparison (or relational) operators
  • boolean operators
Comparison Operators

Comparison (or relational) operators test a relationship between two values and determine if the relationship is true or false.

Operator Relationship Tested
== Equal to
!= Not equal to
< Less than
> Greater than
<= Less than or equal to

By comparing two values or expressions, you test that a relationship (such as "less than") exists between them. If the relationship does exist, the condition is true; if not, it is false. These are the relationships you can test in Java.

You can use relational operators to compare variables, constants and the values of arithmetic expressions. The following table shows the results of using the relational operators to compare different types of expressions.

Be careful when useing relational operator == with Strings or other Compound types (ADTs).

The == operator compares the object references (the addresses of the locations of the objects in memory) and tests them for equality, not the content of the object.

When testing Strings (or other objects) for equality, you should always use the equals() method.
Boolean Operators

Boolean operators AND (&&), OR (||) and NOT (!) take Boolean expressions as operands. Java uses the symbols &&, || and ! for the Boolean operators. By combining comparison operators with Boolean operators, you can make more complex conditions. For example, suppose you want to determine whether your final mark is greater than 90 AND your midterm mark is greater than 70. In Java you would write the expression this way: 

finalMark > 90 && midtermMark > 70

In another example, suppose you wanted to determine whether the midterm level is a 4 OR the final level is a 4. If either the midterm level or the final level equals 4, the condition is True. In Java, you can write the expression like this: 

midtermLevel == 4 || finalLevel == 4

The Boolean AND operator is very helpful for verifying that a given number is between two other numbers.

Say you want to check if a number is between 80 and 100. This will be true if the number is above 80 AND below 100. Use Boolean and comparison operators as follows: 

number > 80 && number < 100
This expression will return true when number is between 80 and 100, and false otherwise.

Similar to arithmetic operators (+, -, *, /, %), Boolean operators have a recognized order of operations. If you required the operations to be evaluated in a different order, use parentheses to change the operator precedence.

  1. NOT (!) is evaluated first
  2. AND (&&) is evaluated next
  3. OR (||) is evaluated last

When combining comparison and Boolean operators, you will need to perform the comparisons first.

Selection Control Structures

You can use a selection control structure when you want the computer to choose between alternative actions. You make a conditional statement that is either true or false.

If the conditional statement is true, the computer executes one statement or set of statements.

If it is false, it executes a different statement or set of statements.

The computer's ability to solve practical problems is an outcome of its ability to make decisions and execute different statements.
A selection structure using flowchart notation

If Statement

The if statement is the fundamental selection control structure that allows the computer program to select certain statement(s) to execute. With it, you can test a relationship and choose a course of action.

The if statement comes in two forms, the if form and the if-else form.

If form
If-Else form
Sometimes you run into a situation where you want to say, “If a certain condition is true, perform some action; otherwise, don't do anything.” The If form of the If statement would be used in this case. Sometimes you run into a situation where you want to say, “If a certain condition is true, perform one action; else perform a different action.” The computer performs just one of the two actions under any given set of circumstances. The If-Else form of the If statement would be used in this case.
The syntax of an if statement is: 
      if ( <condition> ) {
          <statements>
      }
 The syntax of an if-else statement is: 
      if ( <condition> ) {
          <statements #1>
      } else {
          <statements #2>
      }
The must produce a Boolean result. If it is true, the statement(s) are executed. The must produce a Boolean result. If it is true, the statements #1 are executed. If it is false, the statements #2 are executed.

Example 2: If Statement

    if (temperature > 0) {
        Console.print("Above freezing");
    }

Example 3: If/Else Statement

    if (age < 18) {
        Console.print("Too young to vote.");
    } else {
        Console.print("Old enough to vote!");
    }
 
  • The condition must be placed inside parentheses "()"
  • Notice that both forms of the "if" statement DO NOT end with a semicolon!
  • Also note the curly brackets used to denote the code inside the selection structure.
  • The body of a selection statement should be indented.
  • An open curly brace "{" should be placed on the same line as the if statement, and the closing curly brace "}" should be on a separate line and aligned with the if statement
  • If there is an else statement, it should have the closing curly brace "}" and the opening curly brace "{" on the same line

The Else-If Statement

Suppose you wanted to choose from a set of alternatives and not from just two alternatives. An else-if statement could handle the choices.

The syntax of an else-if statement is:

    if ( <condition #1> ) {
        <statements #1>
    } else if ( <condition #2> ) {
        <statements #2>
    } else {
        <statements #3>
    }

Each condition must produce a Boolean result. The FIRST condition to evaluate to true will cause its block of code to execute. If none of the conditions evaluate to true, the code in the else block is execute.

Example 4: Else-If Statement

    if (guess == SECRET_NUM) {		//correct
        Console.print("You guessed it!");
    } else if (guess < SECRET_NUM) {	//too low
        Console.print("Too low.");
    } else {				//too high
        Console.print("Too high.");
    }

Note that if a condition is true then only the statements with that condition are executed. None of the other conditions are tested or executed. This is important to keep in mind while choosing the order in which to create your condition statements! The order of your expressions is important because it can affect what statements are executed.

Nested If Statements

A nested if statement refers to an if statement inside another one. In the example below, The only way the homeroom == 100 condition is evaluated is if gradeLevel is equal to 10, otherwise it is completely skipped. If gradeLevel is equal to 10, then the program will evaluate the code inside this statement.

    if (gradeLevel == 10) {
        if (homeroom == 100) {
            Console.print("You are in Homeroom 100.");
        } else {
            Console.print("Your Homeroom is not listed in Grade 10.");

    }

Repetition Control Structure

The repetition or looping control structure executes the same statement(s) over and over, as long as a condition or set of conditions is satisfied.

If the condition is true, the statement(s) are executed and then the condition is tested again.

If the condition is false, the statements after the loop are executed.

Each execution of a loop is called an iteration.

There are three types of looping statements in Java that you will be introduced to:

  • while
  • do while
  • for
A repetion structure using flowchart notation

While Statement

Do-While Statement

The while statement evaluates a condition and if the condition evaluates to true, it executes one or more statements before looping back to evaluate the condition again. After the condition evaluates to false, the statements are skipped and the loop ends.

The do..while loop is similar to a while loop, except that the statements are executed once before the initial evaluation of the condition. After that, the statements are executed only if the condition evaluates to true.
The syntax of a while statement is: 
    while ( <condition> ) {
        <statements>
    }
 The syntax of do-while statement is: 
    do {
        <statements>
    } while ( <condition> );
The <condition> must evaluate to a Boolean value - either true or false. This condition is checked before any of the statements inside the loop are executed; this means is it possible that the while loop may never execute if the condition is initially false. The <condition> must evaluate to a Boolean value - either true or false. This condition is checked after the statements inside the loop are executed; this means that the do-while loop will always execute at least once.

Example 6: While Statement

    int num = 1;
    while (num < 100) {
        num *= 2;
    }

Example 7: Do-While Statement

    int num;
    do {
        num = Console.readInt(“Enter a number less than 4”);
    } while (num >= 4);
  • The condition must be placed inside parentheses
  • Notice that the while statement DOES NOT end with a semicolon for a while loop, but DOES have a semicolon in a do-while loop!
  • Also note the curly brackets used to denote the code inside the repetition structure.
  • The body of a repetition structure should be indented.
  • The opening curly bracket "{" should be on the same line as the while statement in a while-loop, and the do keyword in a do-while loop.
  • The closing curly bracket "}" is on its own line in a while loop, and is on the same line as the "while" statement in a do-while loop.

A while loop / do-while loop continues until its condition is false. If the condition never become false, the loop is called an infinite loop (continues forever!!)

For Statement

  • The for statement is a loop structure
  • It executes a set of statements a fixed number of times
    • Takes the form: 
        for ( <initialization>; <condition>; <increment> ) {
      <statements>
    }
  • Initialization
    • The initialization is executed once
    • Used to set the value of the counter
    • Usually used to declare the counter variable as well
    • Example: 
      int i = 0;
  • Condition
    • The condition is a Boolean expression
    • Evaluated before each loop iteration
    • When true, the statements inside the loop are executed
    • When false, the program flow continues following the "}"
    • Example: 
      i < 10;
  • Increment
    • After each loop iteration, the increment is executed
    • Example: 
      i += 1;
    • This can also be written as: 
      i++;
    • The increment can be any arithmetic expression that assigns a new value to the variable
    • You can also decrement the variable instead, using 
      i -= 1;
      or 
      i--;

Example 8: For Statement

  for (int i = 1; i <= 10; i++) {
    Console.print(i);
  }
  • The variable i is declared in the initialization portion of the for loop, which means that the use of the variable is limited to the loop (called the scope of the variable).
  • Notice that the "for" statement DOES NOT end with a semicolon, but semicolons are used to separate each part of the "for" statement
  • Also note the curly brackets used to denote the code inside the repetition structure.
  • The body of a repetition structure should be indented.
  • The opening curly bracket "{" should be on the same line as the "for" statement
  • The closing curly bracket "}" is aligned with the "for" statement.

Counters, Accumulators and Sentinels

Many algorithms require counting and summing of values.

Counters Accumulators

Useful for keeping track of the number of times a user enters a value, makes a guess, or types a password.
A program that counts the number of values entered by the user is actually counting the number of loop iterations.

Used for a wide range of reasons such as finding the total of a list of numbers and calculating your score in your favourite game.
A statement like this can be used to implement a counter within a loop: 
numValues += 1;
 A statement like this can be used to sum values as they are entered: 
sumOfValues += newValue;
Each time the statement executes, 1 is added to the current value Each time the statement executes, the value of newValue is added to the current sum
A counter should be initialized when it is declared (set to 0), and then incremented by a fixed amount. An accumulator should be initialized when it is declared

A sentinel value is a special value that acts as a flag to signify that a condition has been met.  Sentinel values are often stored in constants. Sentinels are used in several program structures including if and loops structures.

The following example demonstrates the use of counters, accumulators and sentinels.

Example 9: The Average Calculator Program

Consider a program that allows the user to enter a series of non-zero values. The program will calculate the average (arithmetic mean) of the values entered. In order to do this, the program will have to keep track of the number of values entered.

/*
 * AverageValue.java
 * An application for demonstrating the use of counters, accumulators and sentinels.
 */

package activity2;

import simpleIO.Console;

public class AverageValue {

    public static void main(String[] args) {
        /* Constants and Variables */

        // acts as a flag, or sentinel to signify that the loop should stop
        final int SENTINEL = 0;

        int newValue;  //value entered by user
        int numValues = 0;  //counter
        int sumOfValues = 0;  //accumulator
        double average;

        Console.print("Calculate Average Program");

        /* Input - Get a set of numbers from user */

        newValue = Console.readInt("Enter a value (" + SENTINEL + " to quit):");

        while (newValue != SENTINEL) {
            numValues += 1; // counter
            sumOfValues += newValue; // accumulator
            newValue = Console.readInt("Enter a value (" + SENTINEL + " to quit):");
        }

        /* Guard against run-time error - no calculation if no values were entered */
        if (numValues == 0) {

          Console.print("No average calculated");

        } else {

          /* Processing - Calculate average of numbers entered by user */
          average = (double) sumOfValues / numValues;

          /* Output - Displaying results to user */
          Console.print("Average is " + Console.roundDouble(average, 2));
        }
    }
}
