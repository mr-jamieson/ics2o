Methods

In programming, a larger program can often be divided up into smaller parts. We call these smaller parts subprograms, or in Java, we use the term methods. These smaller parts can be combined to solve the larger problem.

There are many benefits to writing programs using methods:

  1. They allow the programmer to break a large problem into smaller parts, making it easier to solve
  2. They allow the programmer to save time by reusing code. Any time the particular task needs to be completed, the method can be called, instead of repeating the code all over again
  3. They keep programs very well organized, and accomplish each task within small blocks of code
  4. They can be tested using automated unit tests

Definition: A Java method is a collection of statements that are grouped together to perform an operation. When you call the Console.print() method, for example, the system actually executes several statements in order to display a message on the console.

You have already been using methods (Console.print(), Console.readInt(), etc ...), this activity will help you learn how to create methods as well!

A method consists of:

  • A declaration (also called a header or signature): optional modifiers, return type, name and parameters (if any)
  • A body: statements that implement the method

The general form of a method is:

<optional_modifiers> <return_type> <name>(<parameters>) {
    <statements>
}

Example 1: A simple method - sayHello()

Below is a very simple example of a method. Note that this method has:

  • the public modifier to control access level
  • the static modifier to create a class method
  • a void return type (meaning that the method does not contain the keyword return, thus it does not return a value)
  • no parameters (their brackets are empty)
package activity3;
import simpleIO.Console;

public class MethodsExample {

    public static void main(String[] args) {
        sayHello();
    }

    public static void sayHello() {
        Console.print("Hello World");
    }

}
  • Methods must belong to a class and need a descriptive name, that should be a verb (action word) such as findRadius() or sayHello()
  • When calling a method, ALWAYS use brackets after the method name: sayHello()
  • For today, all your method headers will contain the static modifier; this makes it a class method, instead of an instance method. We will learn more about this later!
  • Since Java is a compiled language, it does not matter what order you write your methods in, or whether they appear above or below the main method.

Passing Data to a Method - Parameters and Arguments

When you create a method, you can have it receive any kind of value you want, such as String, int or even objects (ADTs). Sending data to a method is known as passing data.

In order to transfer the data you need to:

  1. pass the data to the method, and
  2. catch the data in the method

When the data is passed to the method, we refer to it as a parameter, and when the data is caught in the method, we refer to it as an argument.

Passing the Data
Method calls pass values to methods.

Data is passed to a method in the statement that calls it. You may have noticed that when calling a method, you must include brackets at the end of the statement. These brackets hold parameters that you can send to the method.

You can send any kind of data (int, float, String, boolean) this way. The data can be a literal statement or a value contained in a variable.
Catching the Data

In order for the data to be received, the method statement must include variables (called arguments) that will “catch” the data. The variables' data types must match the data type of the incoming data.

The variables are placed in brackets after the method's name. The code takes the form:

public void methodName (dataType variableName)

The received value tells the method what kind of data to expect and places it in a variable.

 The methods catch the passed value.
 
  • When calling a method, you can use parameters to pass values into the method.
    • Parameters are passed by enclosing them in parentheses (multiple parameters separated by commas) in the method call
  • When writing a method, you can accept arguments (data that is passed to the method) to be used inside the method (do not use a new declaration)
    • Arguments are accepted into a method by stating both the variable type and the argument name in the method header.
  • The order of your parameters and arguments is important; the order must match both in type and intent.
    • However, argument names do not need to match parameter names

Example 2: Method with parameters

Below is a simple example of a method with parameters and arguments:

package activity3;
import simpleIO.Console;

public class MethodsExample {

    public static void main(String[] args) {
        showSum(2, 3, 10);
    }

    public static void showSum(int a, int b, int c) {
        int sum = a + b + c;
        Console.print("The sum is: " + sum);
    }
}

Note that the method call passes three parameters: the integers 2, 3 and 10.
The method header identifies that those integers will be accepts as arguments, into integer variables named a, b and c.

  • Any variables declared inside a method (including the main method) have a local scope - they cannot be used or accessed outside of the method.
    • This means that methods do not have access to the variables in other methods unless they are sent as parameters
  • It is possible to have more than one method with the same name, but with different parameters. This is called method overloading.
    • The compiler knows which method to used based on the number, type and order of the parameters

Example 3: Overloaded methods

The next example shows a possible implementation of drawing right triangle using a single character.

There are two versions of the method drawBar(), one that which accepts a parameter i into the argument length, the other that also accepts a parameter '#' into the argument symbol.

Since the method call is inside a for loop, the first part will make a total of 5 calls to the method drawBar(int), each printing one more asterisk than the last one. The second part will make 10 calls to the method drawBar(int, char).

package activity3;
import simpleIO.Console;

public class RightTriangle {

    public static void main(String[] args) {
        /* draw a right triangle with base size 5 */
        for (int i = 1; i <= 5; i++) {
            drawBar(i); //parameter: i
        }

        /* draw a right triangle with base size 10 with a custom symbol */
        for (int i = 1; i <= 10; i++) {
            drawBar(i, '#'); //parameters: i, '#'
        }
    }

    public static void drawBar(int length) { //argument: length
        for (int i = 0; i < length; i++) {
            Console.print("* ", false);
        }
        Console.print(); //force newline
    }

    public static void drawBar(int length, char symbol) { //arguments: length, symbol
        for (int i = 0; i < length; i++) {
            Console.print(symbol + " ", false);
        }
        Console.print(); //force newline
    }
}

Methods That Return Values

In the previous examples, you learned how to pass data to a method from a call statement. A method can also send data back to the line of code that called it. This is called returning a value from a method.

In order to return a value from a method, you need to do three things:

  1. specify which kind of value the method returns;
  2. return the value;
  3. create a variable that receives the returned variable.
 Methods can accept and return values.

When writing a method, the second word in the statement describes the returned data type.

You have seen methods with a void return type. This meant that the method did not return a value.

When you do want to return a value from a method, you can replace void with any data type.

Setting a return data type takes the form:

scope ReturnDataType methodName (SentDataType variable)

Where

ReturnDataType can be String, int, double, boolean or other data type.
Returning a Value

To send a value back to the code that called it, you use the command return.

return variable;

will send the variable back to the line of code that called it.

Within the method, you will declare the variable that will be returning the value. The data type of this variable must match the returned data type set in the method's header.
  • A method can return a value
  • Use the <return_type> in the method header to specify int, double, String, etc.
  • Use the return statement at the end of the method body to send a value back to the calling statement
    • Ideally, each method should only contain ONE return statement! This is good style!
  • You should use the value that is returned from the method in some way:
    • Set a variable equal to the method call using = 
      int interest = calculateInterest(amount, rate);
    • Use the method call as an operand in an expression (arithmetic, boolean or String) 
      int term = coefficient * power(base, exponent);
if (nextValue() == 4) { ... }
    • Pass the result as a parameter to another method 
      Console.print("Score: " + calculateScore());
    • Advanced: If the data returned is an ADT (such as String) you can call another method on the output object (called method chaining) 
      String nameInCaps = getName().toUppercase();
  • In Java, a method may only return one value

Example 4: Returning the Sum

Below is a modification of the showSum(int, int, int) from example 2 above, which returns the sum, so main method can output it to the console.

package activity3;
import simpleIO.Console;

public class MethodsExample {

    public static void main(String[] args) {
        int total = calcSum(2, 3, 10);
        Console.print("The sum is " + total);
    }

    public static int calcSum(int a, int b, int c) {
        int sum = a + b + c;
        return sum;
    }
}

In this version of the program, the result of the addition statement is returned to the main method, and assigned to the variable total.

  • Recall: Any variables declared inside a method have a local scope - they cannot be used or accessed outside of the method.
    • Methods can only send information back to where they are called from with a return statement

Parameters and return statements are like a one-way-street. Information flows either to the method (parameters), or back to the code that called the method (return statement)

Example 5: CubeCalculator

The example below shows a method cubeOf(int), which returns the cube of its parameter.

Notice this example includes two methods that return values - the Console.readInt(String) as well as cubeOf(int). Since cubeOf is defined inside the same class as the method call, you do not need to include the class name. However, the readInt() method is located inside the Console class, so you must include its class name for the method call to work.

package activity3;
import simpleIO.Console;

public class CubeCalculator {

    public static void main(String[] args) {
        int num, cube;

        // Input
        num = Console.readInt("Enter a number:");

        // Processing
        cube = cubeOf(num);

        // Output
        Console.print("The cube of " + num + " is " + cube);
    }

    public static int cubeOf(int x) {
        int cube;
        cube = x * x * x;
        return cube;
    }

}

In the previous examples, we created methods that printed directly to the Console. Our goal movingf forward is to limit calls to Console from methods and use the Console class only in the main method. One way to do this is to have our methods return a String, and then this value is printed to the Console.

Example 6: RightTriangle, Revisited

Below is an example of a modified RightTriangle application from example 3. Notice that now the method returns a String instead of using Console.print().

package activity3;
import simpleIO.Console;

public class RightTriangle {

    public static void main(String[] args) {

        int triangleSize = Console.readInt("Enter a size for your triangle: ");
        char symbol = Console.readChar("Enter the symbol you would like to use: ");

        /* draw a right triangle with base size entered by user */
        for (int i = 1; i <= triangleSize; i++) {
            String lineToPrint = createBar(i, symbol);
            Console.print(lineToPrint);
        }
    }

    public static String createBar(int length, char symbol) { //arguments: length, symbol
        String triangle = "";

        for (int i = 0; i < length; i++) {
            triangle += symbol + " ";
        }

        return triangle;
    }
}

One final note about terminology: Methods that have a void return type are also called procedures. Methods that return a value are also called functions.

Program Testing

It is important that a programmer validate his/her program using a full range of test cases before it is implemented. This means that when you are testing your program you should try and see how it will react to a variety of input values and ensure that the results you receive are correct. When creating test cases, you should include both regular inputs, as well as inputs that might cause potential problems in the program.

To manually test a program can be very time consuming, so it can be helpful to use an automated testing strategy to simplify the process. In this activity we will be learning about the JUnit automated testing framework that is used by many programmers to automate their testing.

In order for automated testing to be useful, you must use functions in your programs. The JUnit framework is used for testing methods, but it is not meant for testing the main() method. Recall the general model for designing computer programs: Input - Processing - Storage - Output. Ideally, your methods will contain the Processing portion of your program, while the main method, using the Console class, will handle the Input & Output. This setup will allow you to make use of automated testing.

Creating Automated Tests

To create a Junit test, it is considered best practice to use the name of the class that you are testing, followed by the keyword "Test". In this course, all classes that end with the letters "Test" are considered JUnit tests!

So, if we wanted to create a JUnit test for the CubeCalculator program above, we would name it CubeCalculatorTest.java

Once your JUnit Class is created, you will need to populate it with test cases. For each test case, you must decide what input you will use, as well as the output that you expect to receive. Then call the assertEquals(expected, actual) method provided by the JUnit framework.

  • The expected parameter should be the output that your method should provide (this can be a String, int, double, boolean, etc)
  • The actual parameter should include a call to the method under test (don't forget to include the class name) with the inputs that should produce the output identified in the previous step
  • When testing methods that return double values, a delta parameter is needed. This is the maximum difference between expected and actual for which both numbers are still considered equal

Example 7: JUnit Test for CubeCalculator

The following is a simple automated test for the RectangleCalculator class. Notice that each test scenario (normal behaviour, one input is zero, one input is negative) is placed in its own method, with one or more test cases per method. Also the annotation "@Test" must be placed prior to each method so JUnit knows how to run it correctly.

package activity3;

import static org.junit.Assert.*;
import org.junit.Test;

public class CubeCalculatorTest {

    @Test
    public void testNormalScenario() {
        assertEquals(125, CubeCalculator.cubeOf(5));
        assertEquals(8, CubeCalculator.cubeOf(2));
        assertEquals(1, CubeCalculator.cubeOf(1));
    }

    @Test
    public void testZeroValues() {
        assertEquals(0, CubeCalculator.cubeOf(0));
    }

    @Test
    public void testNegativeValues() {
        assertEquals(-8, CubeCalculator.cubeOf(-2));
    }

}

There are two options for running JUnit tests. The first involves using the JUnit Console Standalone program, provided in your Repl.it. The other option involves manually editing the TestRunner.java program found in your test folder. Your teacher will demonstrate both!

Why is the extra parameter needed when testing methods that return doubles? This is due to the lack of precision in storing double variables, which you may have already noticed in some of your programs.

Usually, the delta argument is a small number that represents the allowable rounding error.

For example, suppose we had a method called calculatePay(int hours, double rate) in an Employee class, which accepts the number of hours worked and the hourly rate, and returns the total pay. The assert statement to test that method could be:

assertEquals(112.50, Employee.calculatePay(15, 7.50), 0.005);

This delta value of 0.005 would allow for rounding (up or down) to the nearest hundredth (2 decimal places of accuracy).

When creating your tests, consider the following suggestions:

  • Test typical values (values you expect the user will enter)
  • Test very large, very small, and both negative and positive values.
  • Test boundary conditions using the extremes of your input domain (maximum, minimum, just inside or outside the boundary)
  • Test all of the branches of your decision structures
  • Test repetition structures for typical and large numbers of iterations, as well as zero iterations
  • Test conditions that should generate an error.

To learn more about JUnit and automated testing, read the JUnit API

Finally, it's important to realize that not all programs can be testing only with automated tests (there will still be a need for some manual testing!), but they are a great asset for designing robust programs. Many programmers adopt a "test-driven development" programming style, where the programmer creates their automated tests before creating their programs. That way they know when their program is complete; it passes all the test cases!


The Math Class

The word "computer" traces its origin back to the 17th century, meaning "one who calculates, a reckoner, one whose occupation is to make arithmetical calculations" (source: Online Etymology Dictionary.) The early computers were large calculating machines. Computers make mathematical calculations vastly simpler and quicker than performing the same calculations by hand.

The Math class contains methods and constants for common mathematical functions and values.

The constants in the Math class are:

The random() function is often used within arithmetic expressions to create random numbers within a specified range.		 The following method will return a random number between a and b (which may include a or b). Use this method when generating a random number between any 2 numbers: 
public static int randomNumber(int a, int b) {
  int highNum = Math.max(a, b);
  int lowNum = Math.min(a, b);
  int range = highNum - lowNum + 1;
  return (int) (Math.random() * range) + lowNum;
}
